## Findings (same list)

| ID | Severity | Location | Claim | Suggested fix |
|---|---|---|---|---|
| SR-B1 | blocker | `src/lib/auth.ts` `getStaffContext` | Staff lookup is `clerkUserId OR email`. When a row matches by verified email but `staff.clerkUserId` is already set to a *different* Clerk user, the code still returns that staff context and only bumps `lastLoginAt` — it never asserts `staff.clerkUserId === identity.clerkUserId`. A second Clerk account with the staff email verified can inherit staff privileges. | Prefer `clerkUserId` match. On email match, auto-link only when `clerkUserId` is null; if set and unequal, deny (or require explicit manager rebind). |
| SR-B2 | blocker | `src/app/api/driver/[token]/route.ts` GET; `src/lib/routes/service.ts` `loadMagicLinkSession` | Magic-link GET returns full stop PII (recipient names, street addresses, maps URLs) for any valid token. `pinRequired` only gates client UX and mutating POSTs (`verify-pin` / `start` / `deliver`). Anyone with the URL can read the roster without the PIN. | Until PIN verified (when `pinRequired`), return only `{ pinRequired, linkId }` (or redacted stops). Load full stops after successful unlock. |
| SR-B3 | blocker | `src/lib/stripe/client.ts` `getStripeMode`; `src/app/api/checkout/mock-complete/route.ts` | Missing/`mock` Stripe secret (or `STRIPE_MODE=mock`) silently selects mock mode. Public `mock-complete` then signs a webhook with the mock secret and can finalize payment for an accessible draft without a real charge. Env schema does not forbid mock in `NODE_ENV=production`. | Fail closed in production unless `STRIPE_MODE` is explicitly `live`/`test` with real keys; gate `mock-complete` behind non-production + mock, and refuse mock mode when `NODE_ENV=production`. |
| SR-M1 | major | `src/app/(admin)/admin/routes/page.tsx`; `src/app/(admin)/admin/routes/[id]/page.tsx`; `src/app/(admin)/layout.tsx` | Routes admin pages render with no `requireAdminPage` / `requirePermission`. Admin layout only wraps chrome when staff has `admin.access`; otherwise it still renders `children`. Any signed-in Clerk customer can open `/admin/routes` UI (APIs still 401/403). | Call `requireAdminPage("admin.access")` on both pages, or enforce once in the admin layout / redirect. |
| SR-M2 | major | `src/lib/payments/webhook.ts` `handleChargeRefunded` / `processStripeWebhook` | `charge.refunded` and `refund.created` share one handler typed as `RefundObject`. Stripe’s `charge.refunded` object is a **Charge** (`amount_refunded`, charge `id`), not a Refund (`amount`, refund `id`). Live `charge.refunded` events can no-op, mis-key idempotency, or apply a wrong amount; only `refund.created` matches the assumed shape. | Branch by event type: parse Charge vs Refund; for charges iterate `refunds.data` (or rely solely on `refund.created`) with correct amount + refund id. |
| SR-M3 | major | `src/middleware.ts` (`AUTH_MODE=dev` branch) | When `AUTH_MODE=dev`, middleware returns `NextResponse.next()` for all matched routes without checking `NODE_ENV`. Dev session API is production-gated, but the edge still skips Clerk `auth.protect()` for `/admin/*` and other non-public paths if mis-set on a prod-like host. | Mirror `getAuthIdentity`: refuse or fail closed when `NODE_ENV===production` and `AUTH_MODE=dev`; never skip protect in production. |
| SR-M4 | major | `src/lib/orders/guest-token.ts` `guestDraftCookieOptions` | Cookie options always set `secure: true`. Default `APP_URL` / `.env.example` use `http://127.0.0.1:3103`. Browsers will not store Secure cookies on `http://127.0.0.1`, so guest draft tokens fail after create (guest checkout broken on documented local URL). | Set `secure` from `APP_URL` protocol (or `NODE_ENV===production`); keep httpOnly + sameSite. |
| SR-M5 | major | `src/lib/http/public-guard.ts` | Rate limits are process-local `Map`s; anonymous clients without a session share one `"anon"` identity. Multi-instance deploys reset counters per isolate; one noisy client can also exhaust the shared anon bucket for everyone. | Shared store (Redis/Upstash) or platform edge limits; avoid a single global anon key (or bind to a non-spoofable identity). |
| SR-M6 | major | `src/lib/orders/finalize.ts` | File is double-spaced (blank line between nearly every source line; ~500+ lines). Violates clean-code readability; inflates tokens; looks like accidental formatting corruption on a critical money/inventory path. | Reformat to normal single-spaced TypeScript; keep logic unchanged; re-run typecheck/smoke on finalize. |
| SR-M7 | major | `src/lib/routes/service.ts` | ~900+ line god file mixing geocode, route build, magic links, PIN throttle, day-of notify, print PDF, nearby suggestions, and reroute/void. Exceeds clean-code >500 / mixed-concerns rule. | Split by concern (`magic-link.ts`, `route-build.ts`, `reroute.ts`, `print.ts`) without behavior change. |
| SR-M8 | major | `src/lib/ops/import.ts`; also `src/lib/ops/repeat.ts`, `src/lib/orders/drafts.ts`, `src/lib/checkout/session.ts`, `src/lib/ops/print-batch.ts` | Multiple modules still exceed the >500-line / mixed-concerns split threshold (import ~670, repeat ~665, drafts ~540, session ~531, print-batch ~513). | Extract parse/classify/commit (import) and similarly peel concerns in the other listed files; keep one orchestration entrypoint each. |
| SR-M9 | major | `src/lib/routes/service.ts` `hashPin`; `src/components/admin/routes-admin.tsx` | 4-digit PIN hashed with unsalted SHA-256 (`pin:${pin}`). Offline brute-force of a stolen `pinHash` is trivial (10k space); UI also defaults create form PIN to `"1234"`. | Slow KDF (scrypt/argon2) with per-route salt; default PIN field empty; keep lockout throttle. |
| SR-m1 | minor | `.env.example` | Template ships concrete-looking secrets (`NEWSLETTER_HMAC_SECRET`, `CRON_SECRET`, `DRAFT_ACCESS_SECRET`, mock Stripe/cron values) instead of placeholders — easy to copy into real deploys. | Use `<set-me>` placeholders; document required vars without default secret material. |

Event idempotency mitigates duplicate ids but not delayed replay of unused event ids. | Reject when `\|now - t\|` exceeds ~300s. |

Historical ORDERS import (`classifyOrderRows` / `commitOrderRow`) is not what the stub/smoke path exercises end-to-end. | Wire repeat/migration smoke through real `ImportKind.ORDERS` stage+commit, or mark stub non-production and stop treating it as migration proof. |
