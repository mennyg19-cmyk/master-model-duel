## Summary

P7 ships two parallel implementations of the package helpers: a season-scoped set in `src/lib/packages/board.ts` + `src/lib/packages/actions.ts` (used by the `[id]/split`, `[id]/stage`, `bulk-stage`, and `regroup` sub-routes) and an **unscoped** set in `src/lib/ops/packages.ts` (used by the top-level `/api/admin/packages` and `/api/admin/packages/[id]` routes). The unscoped set has no `seasonId` filter anywhere, so the top-level routes let any admin read/mutate packages across **all** seasons, including archived ones. The season-scoped helpers in `board.ts` are dead code (no callers).

All P7 routes gate only on `admin.access`, which `STAFF` receives by default (`src/lib/permissions.ts`), so every cross-season issue below is reachable by a low-privilege staff user, not just managers.

Print-batch routes accept a client-supplied `seasonId` (nightly / reprint-group) or `orderId` (reprint-order) with no season membership check, so admins can regenerate PII-laden PDFs for any season or order.

One audit-integrity bug in `bulk-stage` writes phantom audit rows for packages a concurrent transaction already moved.

Dev auth (`AUTH_MODE=dev`, set in the reviewed `.env`) trusts an unsigned `x-dev-user-id` header / `dev_user_id` cookie; if that mode ever reaches a reachable deployment it is a complete auth + authorization bypass.

## Notes

- S-01 is rated major because exploitation requires `AUTH_MODE=dev` to be reachable. If that mode is ever deployed to a non-localhost environment, it becomes a blocker (full auth + authorization bypass with attacker-chosen role).
- S-02 through S-08 all share one root cause: the top-level `/api/admin/packages` and `/api/admin/print-batches` routes wire up the unscoped `lib/ops/packages.ts` / `lib/ops/print-batch.ts` helpers, while the nested sub-routes use the season-scoped `lib/packages/*` helpers. The scoped helpers in `lib/packages/board.ts` are dead code. The fix path is routing-level, not business-logic-level.
- No findings against the `print â‰  shipped` invariant (UR-001, G-001..G-004): `runNightlyPrintBatch`, `reprintFilingGroup`, `reprintOrder`, and `renderArtifactPdf` never mutate `Package.stage`. The audit meta explicitly records `stagesMutated: false`. This is the strongest part of P7.
