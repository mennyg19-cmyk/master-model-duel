# P8 Quality Review — arm-03 (blind)

**Phase:** P8 — Shipping: Shippo, rate margin, labels
**Reviewer:** external quality specialist
**Tree:** `arms/arm-03/workspace`
**EXPECTED:** `shared/phases/PHASE-P8-EXPECTED.md`
**Smoke claim:** 3/3 PASS (`arms/arm-03/results/PHASE-P8-SMOKE.md`)

Scope: correctness, broken flows, stubs, missing smoke, regressions vs EXPECTED. Findings only — no fixes.

## Summary

- **Findings:** 13 (3 high, 6 medium, 4 low)
- **High:** checkout charge and label-purchase quote use different parcels so the margin recorded on the label diverges from what the customer paid; multi-box bin packing is collapsed to a single parcel for the quote so multi-box shipments are mis-rated; label purchase has no idempotency key so a lost DB write after a Shippo `SUCCESS` double-charges the carrier.
- **Medium:** `planPackageShipment` mutates the package as a side effect of a planning call; `refreshTracking`/validate-address/`GET labels` API surface has no UI (orphaned); two `checkoutSnapshot` shapes diverge between prepare and hosted-checkout; `LabelError` is a third error pattern alongside `Result`/`AuthError`/`ActionError`; sequential per-destination Shippo quotes with no caching on the checkout prepare path.
- **Low:** package board shows label actions only as a transient message (no persistent label list, unlike order detail); `Buy label` button is always enabled even when a label already exists; `loadActiveBoxTypes` silently falls back to hardcoded boxes; smoke does not cover multi-box plans, live-mode transaction-amount reconciliation, or void-after-charge fault paths.
