**Notes**
- `lib/notifications.ts` `captureNotification` catches `P2002` and returns `false`. Today `dedupeKey` is the only unique column on `Notification`, so the mapping holds — but the catch will silently treat any future unique-constraint collision as a duplicate. Worth a comment or a check on `meta.target`.
- `lib/email/provider.ts` mock `+failonce`/`+failalways` vs `lib/sms/provider.ts` `[failonce]` use different markers for the same concept. Intentional (phone numbers can't carry `+` tags, per the sms comment) — not a finding, recorded for completeness.
- `sendCampaign` (`lib/email/campaigns.ts:62-73`) issues N individual `captureNotification` calls in a loop rather than a batched `createMany`. Performance, not clean-code — out of scope here.
